    ```javascript
    function resolveTimeOfDayTheme() {
      const h = new Date().getHours();
      if (h >= 4 && h < 9)  return 'ilm';       // Fajr: awakening/knowledge
      if (h >= 9 && h < 17) return 'tawakkul';  // Dhuhr/Asr: effort/perseverance
      return 'shukr';                            // Maghrib/Isha: gratitude/reflection
    }
    ```

    **No-repeat session state (THEME-04, from RESEARCH.md Pattern 7):**
    - `SESSION_STATE_DIR = '/tmp/claude-code-quran-sessions'`
    - `getDisplayedIds(sessionId)`: read JSON array from `SESSION_STATE_DIR/{sessionId}.json`; return [] on any error

    **selectAyah(toolName, sessionId, pluginRoot) function:**
    1. `const theme = resolveTheme(toolName)`
    2. `const displayedIds = sessionId ? getDisplayedIds(sessionId) : []`
    3. Load ALL ayahs matching theme using loadAyah internals — but loadAyah only returns ONE random ayah. To support no-repeat, selectAyah must read fallback.json directly (or call a modified approach). Implementation: read fallback.json at pluginRoot, filter by theme, exclude displayed IDs, pick random from remaining. If remaining pool is empty (all shown), reset and pick from full theme pool.
    4. If no ayah found (null from data layer): return null (DATA-05)
    5. Save the selected ayah's ID: `saveDisplayedId(sessionId, ayahKey)`
    6. Return the ayah object

    **Reading fallback.json directly** (since loadAyah doesn't expose the full filtered pool):
    ```javascript
    const { loadAyah } = require('./load-ayah');  // keep for compatibility
    // For no-repeat: also need direct access
    function loadAyahsForTheme(pluginRoot, theme) {
      try {
        const data = JSON.parse(fs.readFileSync(
          path.join(pluginRoot, 'data', 'fallback.json'), 'utf8'));
        return (data.ayahs || []).filter(a => a.themes && a.themes.includes(theme));
      } catch (_) { return []; }
    }
    ```
