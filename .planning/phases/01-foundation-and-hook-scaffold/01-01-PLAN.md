hooks.json location: `hooks/hooks.json` at repo root.

CLAUDE_PLUGIN_ROOT BUG: `${CLAUDE_PLUGIN_ROOT}` is empty during SessionStart hooks
(GitHub issue #27145, unresolved). The hook command in hooks.json CANNOT rely on
this variable for SessionStart. The session-start.js script itself resolves its own
root via a three-strategy fallback (see RESEARCH.md Pattern 3). hooks.json still
writes `node ${CLAUDE_PLUGIN_ROOT}/scripts/session-start.js` as the command —
this works for non-SessionStart events and the script resolves the root internally
for SessionStart.

Output mechanism: `systemMessage` JSON field on stdout is the ONLY confirmed
working channel for user-visible output. stderr is intentionally suppressed by
Claude Code for exit-0 hooks (issue #12653, closed NOT PLANNED).

async: true is REQUIRED on all display hooks (INFRA-04). Without it Claude blocks.

  <name>Task 2: Create hooks.json with SessionStart hook</name>
  <files>hooks/hooks.json</files>
  <action>
Create `hooks/hooks.json` defining the SessionStart hook.

Requirements:
- INFRA-03: hook command must use `node` as the explicit runtime (not a shell shebang)
- INFRA-04: `"async": true` on the hook — display must not block Claude startup
- INFRA-02: use `${CLAUDE_PLUGIN_ROOT}` in the command path (standard pattern; the script itself handles the empty-variable fallback for SessionStart)
- HOOK-01: the hook event is `SessionStart`, matcher is `"startup"` (fires on fresh sessions only, not resume)

The hooks.json format (from RESEARCH.md Pattern 2):
```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "startup",
        "hooks": [
          {
            "type": "command",
            "command": "node ${CLAUDE_PLUGIN_ROOT}/scripts/session-start.js",
            "async": true
          }
        ]
      }
    ]
  }
}
```

Also add the test hook under a separate SessionStart entry with matcher "startup" pointing to `scripts/test-output-mechanisms.js`. This test hook is for empirical output testing (Plan 03 Task 1 will create the script). Add it commented in the JSON as a disabled entry using `"enabled": false` if the hooks format supports it, OR include it as a second hook in the same hooks array — executor uses judgment here based on what Claude Code's hooks format actually supports. If `enabled` field is not supported, leave it as a second hook that will be manually removed after testing.

The important thing: `async: true` must be present on EVERY hook entry in this file.
  </action>

  </verify>
  <done>`hooks/hooks.json` exists, is valid JSON, contains SessionStart hook with `"async": true`, command uses `node` runtime with `${CLAUDE_PLUGIN_ROOT}/scripts/session-start.js` path.</done>
