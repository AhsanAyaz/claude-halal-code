<!-- fallback.json schema (from RESEARCH.md Pattern 5) -->

Required top-level structure:
```json
{
  "version": "1.0",
  "ayahs": [ ...array of ayah objects... ]
}
```

Required fields per ayah object (DATA-02):
- `arabic` (string): Arabic text with full diacritics where available
- `transliteration` (string): Romanized pronunciation
- `translation` (string): Hilali-Khan English translation text
- `surah_name` (string): English name of the surah (e.g. "Al-Baqarah")
- `surah_number` (integer): Surah number 1-114
- `ayah_number` (integer): Ayah number within the surah

Required tags per ayah object (DATA-01):
- `themes` (array of strings): one or more from ["ilm", "tawakkul", "ihsan", "sabr", "shukr"]
- `time_slots` (array of strings): one or more from ["fajr", "duha", "asr", "maghrib", "isha"]

Distribution requirement (DATA-01):
- 10 ayahs tagged with "ilm" (knowledge/learning)
- 10 ayahs tagged with "tawakkul" (reliance/trust/effort)
- 10 ayahs tagged with "ihsan" (excellence/beauty in action)
- 10 ayahs tagged with "sabr" (patience/steadfastness)
- 10 ayahs tagged with "shukr" (gratitude/thankfulness)

Note: An ayah CAN have multiple theme tags (e.g., ["ilm", "ihsan"]) — it counts toward
all its themes. The constraint is that at least 10 ayahs carry each theme tag.
Total ayah count must be >= 50.

Phase 1 selection: scripts/session-start.js picks a random ayah from those tagged "ilm".
Phase 2 will use the full theme/time_slot routing logic — include all tags now.
